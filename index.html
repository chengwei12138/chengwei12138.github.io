边界值测试

2020-03-03
软件测试

产生原因：1.在输入域的边间或边界附近，常常会发现大量缺陷
2.边界值测试倾向于选择系统边界或边界数据来设置测试用例
四个问题：1.边界在哪里？
2.如何定义边界的邻域？
3.如何选择测试数据？
4.如何设计测试用例？
问题1：寻找系统边界==寻找每个输入条件的边界点
边界点：可能导致被测系统内部处理机制发生变化的点
问题2：给定：a<=x<=b;x的边界点是a,b；
则领域为[a-m1,a+m2],[b-m2,b+m2],m1,m2为很小的值
PS：对于一个输入条件，不同的边界点可以设置大小不同的邻域值
问题3：[a-m1,a+m2],b-m2,b+m2
优化:取a,a+m1,a-m1;
问题4：穷尽测试：在边界及边界附近的邻域内选择所有测试数据构成测试用例
（冗余度：非常高，缺陷定位能力：低）
ps：冗余被测系统对测试用例的内部处理机制相同
基于多边界的测试：所有输入条件同时取边界
（冗余度：不高，缺陷定位能力：低）
ps：缺陷定位能力是多出错的地方进行确定的能力
穷尽有效域测试：针对有效输入中边界附近的数据进行测试
（冗余度：较高，缺陷定位能力：低）
单边界测试：与方案2互补（x1取边界值，x2取有效域内正常值）
（冗余度：较高，缺陷定位能力：高）

hexo+github上传图片到博客

2020-01-16
hexo

在搭建好hexo后如何上传本地文件

1.首先找到你博客目录下的_config.yml文件
2.打开_config.yml文件
3.然后将post_asset_folder: false改成post_asset_folder: true
4.在你的博客目录下就是这个
5.然后git Bash下
6.接着运行 npm install https://github.com/CodeFalling/hexo-asset-image –save 这个是下载插件不然图片是加载不出来的
7.然后输入 hexo new “test”新建个博客测试下
8.在博客里面插入图片的语句这样写
9.最后hexo g -d 部署上去
ps：文件管理一定要做好不然你的git可能都找不到你的博客在哪
